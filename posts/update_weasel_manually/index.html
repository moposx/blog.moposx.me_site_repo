手動更新小狼毫
2021-06-05 21:43 folder [教程]   label 小狼毫   schedule 6 min 10 s.

小狼毫是 RIME 輸入法在 Windows 平臺的前端。然而，由於開發者主力維護的是 macOS 客戶端鼠鬚管，小狼毫遲遲沒有發佈更新。截至寫作本文時，小狼毫最後一次更新是在 2019 年 6 月 22 日，版本 0.14.3，此時 macOS 客戶端鼠鬚管最近一次更新是在 2021 年 2 月 13 日。

在停更接近兩年之際，我認爲有必要寫一篇博文來解釋如何基於官方目前的最新發佈版本來手動更新 RIME 有關組件。

總覽

Weasel 作爲 RIME 前端，內置了核心輸入引擎 librime，利用 OpenCC 進行漢字簡繁轉換，內置了數款輸入方案和符號碼表、詞典等等。這樣我們便知道了如何替換組件以進行更新。

由於 Windows 的安全機制，在 Weasel 正在運行時，我們無法替換相關組件。因此，在執行下列操作時，請先停止“小狼毫算法服務”。

對於本文提到的一些表達，約定如下：

  • 小狼毫的安裝目錄：默認位於 C:\Program Files (x86)\Rime\weasel-{VERSION}。

  • 小狼毫的用戶數據目錄：默認位於 C:\Users\{USERNAME}\AppData\Roaming\Rime。

更新 librime

  1. 前往 GitHub 下載 librime 的 最新發佈版，選擇 win32 版本。

  2. 解壓縮文件，將 dist\rime.dll 拷貝到小狼毫的安裝目錄，在提示文件已存在時，選擇覆蓋。

更新 OpenCC

欲更新 OpenCC，需要更新轉換庫、轉換配置和轉換詞典。

更新轉換庫

使用預編譯版本

  1. 前往 GitHub 下載 opencc 用於 Windows 的 預編譯版本，選擇 Windows (32-bit) 版本。截至本文寫作時，這裏的鏈接不可用。如果讀者遇到該問題，請自行編譯 OpenCC，相關操作將會稍後進行說明。

  2. 因爲 OpenCC 預編譯版本不可見，我也不知道文件結構，讓我們快進到手動編譯 OpenCC。

手動編譯 OpenCC

  1. 從發佈頁面中下載最新的穩定版，選擇 “Source Code (zip)”，將其解壓縮到合適路徑。

  2. 安裝 Visual Studio（筆者使用 Visual Studio 2019）和 CMake （須在 PATH 中）。

  3. 進入 OpenCC 源代碼目錄，雙擊 build.cmd，等待編譯完成（完成後編譯輸出窗口會自動關閉）。

  4. 進入 build\bin 目錄，將 opencc.dll 拷貝到小狼毫的安裝目錄。

更新轉換配置和詞典

使用預編譯版本

（略）

手動編譯 OpenCC

  1. 編譯過程不再複述。

  2. 進入小狼毫安裝目錄，進入 data，備份 opencc 目錄，然後清空其內容。

  3. 拷貝 OpenCC 源碼目錄的 build\data 下所有 .ocd2 文件到 opencc 目錄。

  4. 拷貝 OpenCC 源碼目錄的 data\config 下所有 .json 文件到 opencc 目錄。

更新 rime-prelude

  1. 前往 rime-prelude 的 GitHub 項目倉庫，選擇 "Download ZIP"，將其解壓到合適的路徑。

  2. 將壓縮包內所有 .yaml 文件拷貝到小狼毫安裝目錄的 data 下，並覆蓋同名文件。

更新 luna-pinyin

  1. 前往 luna-pinyin 的 GitHub 項目倉庫，選擇 "Download ZIP"，將其解壓到合適的路徑。

  2. 將壓縮包內所有 .yaml 文件拷貝到小狼毫安裝目錄的 data 下，並覆蓋同名文件。

更新 rime-essay

項目倉庫，選擇 "Download ZIP"，將其解壓到合適的路徑。

  1. 將壓縮包內的 essay.txt 文件拷貝到小狼毫安裝目錄的 data 下，並覆蓋同名文件。

補全遺失文件（可選）

這一步是非必需的，如果讀者不是全新安裝小狼毫並更新組件，那麼應該不用考慮這些操作。

如果是全新安裝小狼毫，直接替換組件之後，小狼毫的設置菜單項是不起作用的，原因是用戶目錄下沒有對應的配置文件，然而由於組件更新的緣故，小狼毫沒有自行創建這些文件，故出現故障。要解決這個問題，只需要進入小狼毫的用戶數據目錄，創建兩個空白文件 default.custom.yaml、weasel.custom.yaml 即可。

後續操作

重啓“小狼毫算法服務”即可。如有錯誤，請重新部署。
